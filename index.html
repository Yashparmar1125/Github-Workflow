4. Daily Workflow

Step 1: Start Fresh

Always pull the latest code before starting.

git checkout dev
git pull origin dev

Step 2: Create Branch

Create a specific branch for your task.

git checkout -b feature/your-feature-name

Step 3: Work & Commit

Code locally. Commit often using semantic messages.

git add .
git commit -m "feat: add user login form"

Step 4: Push

Push your branch to GitHub.

git push origin feature/your-feature-name

5. Commit Standards

We use Semantic Commits. This makes reading history easier and looks professional to recruiters.

Type Description Example
feat A new feature feat: add dark mode toggle
fix A bug fix fix: resolve null pointer on login
docs Documentation only docs: update readme instructions
style Formatting, missing semi-colons style: format home.js

7. Conflict Resolution

Conflicts happen when two people edit the same line of code. Don't panic. Follow this protocol:

  1. Switch to Dev & Update:
    git checkout dev && git pull origin dev
  2. Go back to your branch:
    git checkout feature/your-branch
  3. Merge Dev into your branch:
    git merge dev
  4. Fix the code: Open VS Code. Look for HEAD<<<< markers. Decide which code stays.
  5. Commit the fix:
    git add . && git commit -m "fix: resolve merge conflicts"
"Conflicts are just code trying to hold hands but missing."
